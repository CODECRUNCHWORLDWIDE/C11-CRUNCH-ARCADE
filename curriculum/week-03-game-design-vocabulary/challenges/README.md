# Week 3 — Challenges

The exercises drill the systems. **Challenges stretch you toward shippable artefacts.** This week's challenge is *capturing the work*, not writing more code — game-feel work is invisible if you don't record before and after.

## Index

1. **[Challenge 1 — Juice-Comparison Video](challenge-01-juice-comparison-video.md)** — record a 30-second side-by-side "before juice / after juice" capture of your Week 2 brick-breaker. The same gameplay, same window size, same audio, in two versions. (~120 min including recording, editing, and the 200-word write-up.)

Challenges are optional. If you skip this one you can still pass the week — but the comparison video is the single most useful portfolio artefact from the entire phase-1 of this course. Every game-feel essayist worth reading (Vlambeer, Mark Brown, Jan Willem Nijman) makes their argument with a side-by-side video. Yours can too.

## How to work the challenges

- Read the spec end-to-end before you record a single frame.
- **Capture the "before" first.** It's tempting to record the polished version and reverse-engineer a "before" by stripping things out — but the before will look fake. Capture before juice exists. Commit. Then juice.
- Use the same input sequence in both takes. Players notice when "before" and "after" play different ways. Use the same launch pattern, same paddle path, same final score.
- Commit the video file (or an external URL in `README.md`) to your portfolio repo. Treat it as a deliverable.

Challenges live under your portfolio repo at `c11-week-03-<yourhandle>/challenges/challenge-01/`. The mini-project repo is separate.
